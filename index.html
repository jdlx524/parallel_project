Summary

We aim to implement a parallel cloth simulation using a Mass-Spring model. The simulation will run efficiently on GPUs using CUDA to accelerate computations. We will explore different parallelization strategies to optimize performance and scalability.

Background

Cloth simulation is widely used in gaming, animation, and physical simulations. It involves computing forces for each mass point, integrating movement over time, and handling collision detection. A parallel implementation is necessary to handle high-resolution cloth in real-time.

The Challenge

The main challenges include optimizing memory access patterns, ensuring proper load balancing across threads, reducing synchronization overhead, and efficiently handling self-collision detection.

Resources

We will use CUDA-enabled GPUs for acceleration, Pygame for visualization, and existing physics-based references on cloth simulation. Our implementation will build on the starter code developed in Python.

Goals & Deliverables

  • Implement a parallelized Mass-Spring Model for cloth simulation.
  • Optimize data access using shared memory and coalesced memory access.
  • Explore different work scheduling strategies (static vs. dynamic).
  • Visualize simulation results in real-time.
  • Evaluate performance compared to a CPU-based implementation.

Platform Choice

We will use CUDA for parallel computation due to its ability to handle large-scale physics-based simulations efficiently. The computations map well to the GPU thread model, and memory access optimizations will be explored.

Schedule

  • Week 1: Implement basic Mass-Spring simulation (serial version).
  • Week 2: Parallelize computation using CUDA.
  • Week 3: Implement collision detection and load balancing.
  • Week 4: Optimize memory access and benchmark performance.
  • Week 5: Final testing, report writing, and visualization improvements.
